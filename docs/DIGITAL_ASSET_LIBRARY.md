# Digital Asset Library

## In plain English (start here)

The **Digital Asset Library** at **`/admin/assets`** (Admin → Content → "Asset
Library") is one internal catalog of every piece of **generated media** in the
app — so you never have to hunt across separate tools to find a video, poster,
or caption file.

It shows each asset with a **poster preview**, its **source**, **duration**, the
**files it's made of** (mp4 / poster / captions, each openable), whether it's
**public or internal**, and **where it's used**. It's **read-only** (it catalogs
and links to media; it never edits or deletes it) and **filterable** by source
with a search box.

## What it catalogs

| Source | What it is |
|---|---|
| Training videos | The deeper instructional videos in the Video Library |
| Feature walkthroughs | The Tutorial Center's per-feature recordings |
| Video Studio | Assets generated by the AI video department (best-effort; empty when its store has none) |

Brand assets, blog→social creative, and share-card images can be added later —
see "Adding a source" below.

## How it's wired

```
/admin/assets  (server page, read-only)
  └─ buildAssetLibrary()                ← lib/admin/asset-library.ts
       ├─ collectLibrary()              ← getLibraryItems() (recorded items)
       └─ collectVideoStudio()          ← getRepo().listAssets() (best-effort)
  └─ AssetLibraryBrowser (client)       ← filter + search + preview grid
```

- **Registry-derived, not a filesystem crawl** — it reads the same registries
  the product uses (the library/tutorial video catalogs and the Video Studio
  store), so a new recording or generated asset appears here **automatically**.
- **Self-healing** — the Video Studio collector is wrapped in try/catch, so the
  catalog never breaks if that store is unavailable.

## Adding a source

Add one collector function in `lib/admin/asset-library.ts` that returns
`AssetRecord[]`, and push it into `buildAssetLibrary()`. It will appear in the
grid and the stats with no UI changes. Keep records honest: only list media that
actually exists, and set `recorded`/`public`/`placeholder` truthfully.
